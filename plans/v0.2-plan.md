# v0.2.0 Execution Plan

Source: due-diligence report, 2026-04-20. Target: v0.2.0 shipping with the full action plan landed.

## Ground rules (non-negotiable)

1. **Every worker owns an exclusive file tree.** Workers MUST NOT edit files outside their tree. If they need cli.rs / mod.rs / agent_info.rs / main.rs changes, they write snippet files under `plans/cli-snippets/<worker-id>/` and the lead integrates.
2. **Split files that grow large.** When a worker adds meaningful surface to an existing module (`agents.rs`, `voices.rs`, `music.rs`, `phone.rs`), they convert that file into a directory `src/commands/<name>/{mod.rs, <subcommand>.rs, ...}` — one submodule per user-facing action. No file over ~400 lines.
3. **Framework invariants apply everywhere.** JSON envelope, semantic exit codes (0/1/2/3/4), stdout = data only, stderr = errors/progress, errors include `{code, message, suggestion}`, no stdin prompts, `--help` / `--version` exit 0.
4. **Every new command gets an integration test.** Tests live in `tests/<feature>_<kind>.rs`. Use `wiremock` for HTTP-level regressions (see `tests/http_regression.rs` for the pattern).
5. **Workers DO NOT commit.** The lead commits each logical chunk after integration.

## Handoff protocol

Each worker creates exactly one directory under `plans/cli-snippets/<worker-id>/` containing:

- `cli.rs` — the clap struct / enum additions (new `Commands::` variants, new subcommand enums)
- `dispatch.rs` — the `match` arms to wire into `src/main.rs`
- `agent-info.json` — JSON fragments to append to `src/commands/agent_info.rs`
- `mod.txt` — one `pub mod <name>;` line per new module to append to `src/commands/mod.rs`
- `NOTES.md` — what they built, what they tested, what's open, any new dependency they need in Cargo.toml

The lead integrates all snippets in a single phase before review.

## Phase order

1. **Phase 0** (done by lead): write plan + protocol doc + commit.
2. **Phase 1** (parallel, 9 agents): workers implement in their exclusive trees. Max ~15 min wall clock.
3. **Phase 2** (lead): integrate all snippets into `src/cli.rs`, `src/main.rs`, `src/commands/mod.rs`, `src/commands/agent_info.rs`. Build + test. Commit.
4. **Phase 3** (parallel, 3 reviewers):
   - Codex (GPT-5.4 xhigh): Rust idioms + API correctness vs SDK
   - Gemini (auto): architecture + test coverage
   - Claude general-purpose: framework-compliance re-audit + agent-info contract
5. **Phase 4** (lead): apply review fixes. Build + test. Commit. Repeat phase 3+4 if any reviewer flagged shipping-severity issues.
6. **Phase 5** (lead): update README, CHANGELOG, bump to v0.2.0, tag, commit, push.

## Autonomous continuation

At every phase boundary, lead calls ScheduleWakeup with the same `/loop` prompt so a session drop doesn't stall progress.

## API endpoint references

Per-worker endpoint mappings live in each worker's prompt. Ground truth: `elevenlabs-python` repo (raw_client.py files) and `https://elevenlabs.io/docs/api-reference`.
